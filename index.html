바이브코딩을 시작할 수 있도록 친절한 가이드를 만들어주신 권상님, 그리고 이런 적극적인 논의의 자리를 마련해주신 종목님께 감사드립니다.

이 페이지는 비전공자가 처음으로 바이브코딩을 경험한 후기를 정리한 것으로, 전문적인 튜토리얼이 아닌 개인의 경험 공유임을 밝힙니다.

Step 01
나도 할 수 있을까?
전공자가 아니어도 괜찮아요. 바이브코딩의 본질은 AI와 대화하며 만드는 것입니다.
  • 코드의 모든 줄을 이해할 필요가 없어요. 작동하는 결과물을 먼저 만들고, 궁금한 부분만 AI에게 물어보면 됩니다.
  • AI는 맥락을 주면 줄수록 더 잘 도와줍니다. 구체적으로 시키는 연습이 전부예요.
  • 첫 프로젝트는 "주말 안에 완성 가능한 것"이어야 합니다. 크게 잡으면 중간에 지쳐요.

Step 02
복잡한 설치 없이 시작
바이브코딩 입문자에게 필요한 도구는 딱 3가지입니다.
  • Cursor: AI 기능이 내장된 코드 에디터. cursor.com에서 다운로드.
  • GitHub Desktop: 버전 관리 도구. 터미널 없이 버튼 클릭으로 끝. desktop.github.com에서 다운로드.
  • 브라우저: 이미 있는 크롬/엣지/사파리. HTML 파일을 더블클릭하면 바로 실행됩니다.
  • 터미널, Node.js, 각종 CLI는 지금은 필요 없어요. 익숙해진 뒤에 추가하세요.
💡 도구는 필요할 때 하나씩 추가한다. 처음부터 다 깔면 지친다.
📖 용어 설명
터미널마우스 없이 텍스트 명령어를 입력해 컴퓨터를 조작하는 검은 화면. 개발자들이 주로 쓰지만 바이브코딩 입문 단계에선 필요 없어요.
Node.jsJavaScript를 브라우저 밖에서도 실행할 수 있게 해주는 프로그램. 백엔드·서버 개발에 필요하지만 HTML 단일 파일 프로젝트엔 불필요합니다.
CLICommand Line Interface. 터미널에 텍스트 명령어를 쳐서 프로그램을 실행하는 방식. GUI(버튼 클릭)의 반대 개념입니다.

Step 03
첫 프로젝트는 작게, 구체적으로
1–2시간 안에 완성 가능한 스코프로 잡으세요.
  • 추천: 간단한 퀴즈 앱, 할 일 목록, 카운트다운 타이머, 개인 포트폴리오 한 페이지
  • 비추천: 여러 사용자 로그인이 필요한 앱, 실시간 채팅, 복잡한 대시보드
  • "내가 실제로 쓸 만한 것"을 고르면 완성 동기가 강해집니다.
  • 처음부터 완벽할 필요 없어요. 기능은 나중에 하나씩 추가하면 됩니다.

4
Step 04
AI에게 잘 시키는 법
막연한 요청은 막연한 결과물을 낳습니다. 3요소를 명확히 전달하세요.
  • 무엇을: "퀴즈 앱" ✗ → "4지선다 퀴즈 앱, 10문제 풀고 점수 표시" ✓
  • 어떻게: 색상, 레이아웃, 동작 방식을 구체적으로
  • 기술 스택: "index.html 파일 하나에 HTML, CSS, JS 다 넣어줘, 외부 라이브러리 없이"
  • 예시 데이터도 같이 주면 좋아요. 샘플 문제, 샘플 디자인 등
💡 프롬프트의 질이 결과물의 질을 결정한다
💬 프롬프트 비교 예시
❌ 이렇게 하면 결과가 애매해요
👤 "퀴즈 앱 만들어줘"
🤖 어떤 주제인가요? 몇 문제인가요? 언어는요? 디자인 스타일은요?...
✅ 이렇게 하면 바로 완성 코드가 나와요
👤 "4지선다 한국어 상식 퀴즈 앱. index.html 하나에 HTML/CSS/JS 전부. 외부 라이브러리 없이. 10문제 풀면 점수 표시. 어두운 배경, 민트색 포인트."
🤖 네! 바로 완성 코드 작성해드립니다 ✨
5
Step 05
더블클릭이면 끝
HTML 파일은 별도 설치 없이 바로 실행됩니다.
  • AI가 만든 index.html 파일을 더블클릭하면 브라우저에서 열립니다.
  • 처음 작동하는 순간의 짜릿함이 바이브코딩의 원동력입니다.
  • 마음에 안 드는 부분은 AI에게 바로 "이 부분 이렇게 바꿔줘"라고 수정 요청하면 됩니다.
  • 에러가 나도 당황하지 마세요. 에러 메시지를 그대로 AI에게 던지면 대부분 해결됩니다.
💡 에러는 실패가 아니라 AI에게 물어볼 재료다

6
Step 06
세이브 포인트의 중요성
작동하는 상태는 반드시 저장해두세요. 게임의 세이브 포인트와 같습니다.
  • GitHub Desktop에서 버튼 클릭 몇 번이면 끝. 터미널 명령어 몰라도 됩니다.
  • 기능 하나 추가할 때마다 커밋하는 습관을 들이세요.
  • AI가 코드를 망가뜨려도 이전 커밋으로 되돌아갈 수 있습니다.
  • 커밋 메시지는 한국어로 대충 써도 괜찮아요. "로그인 버튼 추가" 이 정도면 충분합니다.
💡 잘 된 순간을 바로 커밋하라. 미래의 나에게 주는 선물이다.
📖 용어 설명
커밋(Commit)현재 코드 상태를 스냅샷으로 저장하는 것. 게임의 세이브 포인트와 같아요. 언제든 이 시점으로 되돌아올 수 있습니다.
레포지터리코드 파일들을 저장·관리하는 폴더. GitHub에서는 줄여서 "레포(repo)"라고 부릅니다.

7
Step 07
배포하고 세상에 보여주기
혼자 쓰는 걸 넘어, 링크로 공유하는 단계입니다.
  • Vercel을 사용하면 GitHub에 올린 코드를 버튼 클릭 몇 번으로 배포할 수 있습니다.
  • 배포 후에는 yourproject.vercel.app 같은 주소가 생겨서 누구에게나 공유 가능합니다.
  • GitHub에 Push할 때마다 자동으로 업데이트됩니다.
  • 동료에게 링크 하나로 보여줄 수 있고, 모바일에서도 바로 접속 가능합니다.
💡 공유 가능한 링크가 있는 순간, 당신은 개발자입니다.
📖 용어 설명
배포(Deploy)내 컴퓨터에만 있던 코드를 인터넷에 올려서 다른 사람도 접속할 수 있게 만드는 것. 식당으로 치면 "오픈"하는 것과 같아요.
Push내 컴퓨터의 커밋(저장된 코드)을 GitHub 서버로 업로드하는 동작. GitHub Desktop에서 버튼 한 번으로 됩니다.
VercelGitHub과 연결해 웹사이트를 자동으로 배포해주는 무료 서비스. Push할 때마다 자동으로 사이트가 업데이트됩니다.
